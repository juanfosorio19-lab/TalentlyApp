## 🔴 Bug confirmado

### `material-symbols-outlined` en lugar de `material-symbols-rounded`

**Severidad**: Alta — los iconos aparecen como **texto literal** ("arrow_back", "expand_more", "search", "mail") en lugar de los símbolos correspondientes. Afecta usabilidad y look & feel.

**Causa raíz**: el proyecto carga la fuente `Material Symbols Rounded Variable` (via `@fontsource-variable/material-symbols-rounded` en `package.json`), pero 9 archivos usan la clase CSS `material-symbols-outlined`. Esa clase no tiene `font-family` mapeada a la fuente cargada, así que cae al fallback `Inter` y renderiza el nombre del icono como texto.

**Archivos afectados** (verificado con grep):

| Archivo | Tipo |
|---|---|
| `src/views/public/FAQView.jsx` + `.css` | JSX + CSS |
| `src/views/public/PrivacyView.jsx` | JSX |
| `src/views/candidate/CompanyPublicProfileView.jsx` + `.css` | JSX + CSS |
| `src/views/company/CandidatePublicProfileView.jsx` + `.css` | JSX + CSS |
| `src/views/company/CompanyProfileCreatedView.jsx` + `.css` | JSX + CSS |

**Fix sugerido**: replace global `material-symbols-outlined` → `material-symbols-rounded` en los 9 archivos. Es 1 commit limpio.

**Evidencia**:
- `document.fonts` lista solo `Inter` y `Material Symbols Rounded Variable` cargadas (no `Outlined`).
- Computed style de un `<span class="material-symbols-outlined">expand_more</span>` reporta `font-family: Inter, -apple-system, ...` (NO la fuente de iconos).

### Suite 6 — Dark mode (vistas públicas)

Probadas con `document.documentElement.setAttribute('data-theme', 'dark')` + `localStorage.setItem('talently_dark_mode', 'true')`:

- **WelcomeView** (`/`): textos blancos sobre fondo gradient dark. Heading "Talently", subtítulo, CTAs y footer legibles. ✅
- **LoginView** (`/login`): tarjeta con fondo `--surface` oscuro, inputs visibles, botón primary correcto. ✅
- **RegisterView paso 1** (`/register`): cards de Soy Candidato / Soy Empresa con fondos oscuros y bordes correctos. Iconos visibles. ✅
- **FAQView** (`/faq`): ⚠️ Vista funciona pero los iconos salen como texto literal (bug arriba).

> Nota: Inicialmente reporté `WelcomeView` como bug en dark mode (heading invisible), pero fue race condition en mi setup — al setear `data-theme` y navegar inmediatamente, AppContext lee `localStorage` al montar y sobrescribe. Tras setear ambos (atributo + localStorage), la vista se ve perfecta. **Sigue siendo válido el aprendizaje**: el toggle de dark mode en producción depende exclusivamente del flag en `localStorage`. Confirmar en QA manual sec. 1.2 que el switch desde Settings persiste.
