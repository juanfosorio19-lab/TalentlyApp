## Cambios aplicados en esta sesión

### Issue #1 — useSwipeProfiles.js
Cambio defensivo: convertir `swipedIds` (que podía ser undefined) en un `Set` con fallback a array vacío. Mejora también performance del `.filter()` (Set lookup O(1) vs Array.includes O(n)).

### Issue #2 — Chat.jsx
Manejo explícito de errores en `db.matches.get()` y `db.profiles.getById()`. Si fallan, se setea `otherProfile = { full_name: 'Usuario' }` para que la vista no quede en blanco.

### Issue #3 — Charts tooltips
- Nuevas variables en `variables.css`: `--chart-tooltip-bg`, `--chart-tooltip-text`, `--chart-point-border` (con valores invertidos en dark mode).
- `BarChart.jsx` y `LineChart.jsx` ahora leen estas variables via `getPropertyValue()` siguiendo el patrón ya establecido para `--primary`, `--text-secondary`, etc.

### Issue #4 — CompanyFiltersView
Alineadas todas las classNames a `fv__*` para que coincidan con `FiltersView.css` (que ya importaba). Antes la vista renderizaba sin estilos. Cambios:
- `filters-view` → `fv`
- `filters-view__header` → `fv__header`
- `filters-view__back` → `fv__close`
- `filters-view__title` → `fv__title`
- `filters-view__clear` → `fv__reset`
- `filters-view__scroll` → `fv__scroll`
- `filters-section__label` → `fv__section-label`
- `filters-chips` → `fv__chips`
- `filters-chip` → `fv__chip` (+ `--active`)
- `filters-select` → `fv__select`
- `filters-view__footer` → `fv__footer` (+ agregado `fv__btn-clear` para parity)
- `filters-view__apply` → `fv__btn-apply`
